경진대회 · 36분 — 기술 탐색
각자 시도한 모델 공유
  • L2제안자정규화하면 점수 오르지만, 데이터를 안 뭉개고 다 살리니 성능 급등 → 해커톤에선 버린 데이터도 살리자.
  • L2제안자2024 딸기 생육정보로 작물 그룹핑 → 최종 착과수 예측. Life2100에서 영감받은 berry2vec(인코더-디코더) 설명.
  • AI책임Life2100은 데이터가 많이 필요하다는 우려. 대안으로 부스팅 트리(GBT)가 센서 숫자 데이터와 궁합 좋다 소개, LSTM과 앙상블로 인과 보완.
  • 공지한국농업기술진흥원 결재 지연으로 발표 지연(19시 이후 통보). 1분 단위 센서 표준·피밴드 제어 언급.
  • AI책임해커톤 당일 중기부 사업으로 불참 통보(오전만 가능, 원격 지원).
경진대회2 · 60분 — 도메인 지식 대방출
전문가가 딸기 재배 전체를 설명
  • PM모델을 다수확형·품질형·절충형 3종으로 나누자. 현재 생육 위치 측정 모델이 필요.
  • 전문가잎 개수(5~6일 1장)로 단계 진단, 화방 구조(정화방·액화방), 적산온도 600℃=수확, 영양/생식생장·초세, 함수율, 배액 EC/pH·질소 형태.
  • 전문가교과서 온도(주25/야10) 반박 → 동화산물·순광합성 기반 야간온도 제어. VPD 0.8 기준, 포화점(550W·750ppm·25·35℃).
  • 전문가엽병 각도 35~45°, LAI 2.5·적엽, 수정·벌 활동, 병해충(저온다습/고온저습) 설명.
  • 이벤트회의 중 합격 메일 도착(합불 명시 없어 확인 중).
  • PM이미지 기반 현재위치 진단 + 여러 모델 합치는 멀티모델 구조로 정리.
  • 맥락강조자세부 태스크 의미가 맥락 따라 달라짐 → 맥락 반영 필요 제기.
경진대회3 · 30분 — 환경제어와 마무리
"제어를 모르면 진다"
  • 전문가1400명 단톡 경험담: 최다 실패 원인 = 작물 생육·환경제어 이해 부족. 농업 전문가↔개발자 소통이 승부.
  • 전문가겨울 습도: 최소 난방으로 문 안 열고 습도↓, 천창 미세제어, PI(P밴드) 주기별, 조조가온. 에너지=점수.
  • PM예측만이 아니라 제어까지 준비 필요 인식. 룰베이스로 케이스 대응.
  • 맥락강조자맥락(지속가능성·수익)을 기획 축으로. 별도 정리해 피드백 요청하기로.
  • 질문자"온·습도·계절 같으면 착과수 같나?" → 전문가: 화방 차수·착과 스트레스로 달라진다(똑같은 수확량 유지 불가).
  • 행정본선 합격 확정 판단(외부유출 금지 문건 = 대상자 한정). 등록 마감 7/5 오전 10시 — 팀명·소개·슬로건.
03

주요 쟁점 — 입장 대조

의견이 갈리거나 조율된 지점.

쟁점 1 · 어떤 모델 계열로 갈까

L2제안자Life2vec 기반 berry2vec — 작물/센서/온실을 "서사"로 보고 생애주기 학습. 수집 빈도·서사 중심점을 조절하면 데이터 확보 가능.
AI책임Life2100은 LLM 계열이라 데이터가 대량 필요. 예외 케이스에서 제대로 동작할지 미지수. 표/센서 데이터엔 부스팅 트리+LSTM 앙상블이 범용적.
조율: berry2vec를 배제하진 않고 같이 테스트. 단 실용 축은 트리+LSTM 앙상블. AI책임은 "다솜 아이디어가 최신일 수 있다"며 존중.

쟁점 2 · 하나의 큰 모델 vs 멀티모델

PM전체 흐름을 한 모델에 학습시키면 결과는 좋을 수 있으나 데이터-모델 의존이 과도. 데이터 부족 시 성능 붕괴 → 태스크를 잘게 쪼갠 멀티모델로.
맥락강조자세부 태스크에 집중하면 맥락(배경)에 따라 결과 의미가 달라지는 문제를 놓친다. 맥락 반영을 기획에 넣자.
정리: 멀티모델 + 룰베이스로 가되, 맥락 축은 맥락강조자가 별도 문서로 정리해 다음 회의에 반영.

쟁점 3 · 데이터 정규화

L2제안자동일 데이터가 뭉쳐 있음(더미 형식 추정). 정규화로 뭉개지 말고 다 살려야 점수 급등.
합의: 해커톤에선 정규화 최소화, 원본 보존 방향.

쟁점 4 · 예측이면 충분한가, 제어까지 가야 하나

전문가AI가 아무리 정밀 예측해도 실제 환경 제어(습도 배출·천창·최소난방·조조가온)를 모르면 소용없다. 최종 결과물은 열매.
PM동의. 예측 + 제어 케이스·룰까지 준비. 경영적 의사결정(비용) 모델도 추가.
확정 방향: 환경 예측 + 제어 + 비용 최적화를 다 포함. 심사가 "작물 키우는 능력"을 본다는 전제.
