A

영양생장 ↔ 생식생장 완전 정복

딸기 재배 전체가 이 저울 하나를 매일 기울이는 게임이다.

식물이 광합성으로 번 에너지를 어디에 투자하느냐의 문제. 잎·줄기에 투자하면 영양생장, 꽃·열매에 투자하면 생식생장. 딸기가 특별히 어려운 이유는 출뢰기 이후 이 둘이 동시에 진행되고, 재배자가 거의 매일 한 번씩 저울을 왔다 갔다 기울여야 하기 때문이다. "오늘은 영양, 내일은 생식"은 되지만 "이번 달은 영양"처럼 길게 구분할 수 없다.

왜 매일 진동시켜야 하나

딸기는 새 잎이 빨리 나와야 그 사이에 숨어 있던 꽃눈이 밖으로 빨리 나온다(화방 출현 간격 = 생산량). 그런데 생식생장으로만 계속 두면 잎이 안 자라 꽃눈이 못 나온다. 반대로 영양생장으로만 두면 잎만 무성하고 열매가 부실. 그래서 잎도 꺼내고 열매도 키우려면 매일 저울질이 필요하다.

같은 결과(영양/생식)를 만드는 방향이 레버마다 정해져 있다. 여러 레버를 같은 방향으로 밀면 강하게 유도된다.

영양생장이 필요할 때

  • 잎이 작고 부족 (LAI 아직 2.5 미만)
  • 신엽 출현이 느려짐
  • 초세가 약해 스트레스에 취약
  • 겨울 낮 — 자연히 유리한 조건

생식생장이 필요할 때

  • 잎만 무성, 열매 부실(과영양)
  • 수확기 진입 — 품질(맛) 끌어올리기
  • 초세가 과해 웃자람
  • 겨울 밤 — 자연히 유리한 조건

겨울의 자연 리듬과 개입

  • 낮 = 영양생장 유리, 밤 = 생식생장 유리. 겨울엔 이 리듬이 자연히 생겨서 특별히 안 건드려도 왔다갔다 진행된다.
  • 하지만 일장이 계속 짧아지면 식물이 "겨울"로 인식 → 생식생장으로 과도하게 치우침 → 잎 작아지고 열매만 충실 → 결국 세력 붕괴.
  • 개입: 아침·저녁·밤 보광(전조)으로 낮 길이를 늘려 영양생장으로 되돌린다. 경계 밝기 ≈ 2000 lux. 살짝 불만 켜도 식물은 "낮"으로 인식.
목표 상태 — 초세 강 × 영양생장 약

초세(세력)는 강하게 유지하되 영양생장은 약하게 가면, 세력이 좋아 복합방(꽃대가 쌍으로 갈라짐)이 나온다 = 생산량 극대화. 유리온실이 비닐하우스보다 생산성 낮은 건 습도 관리 부실로 이 상태를 못 만들어서다. → 정밀 VPD 관리가 승부처.

B

동화산물 배분 — 야간온도의 과학

"낮에 번 에너지를 열매에 줄까 잎에 줄까"를 밤 온도로 결정한다. 심사에서 실력 차가 나는 지점.

3단 논리

1) 순광합성 = 총광합성 − 호흡 소모. 배분 가능한 실제 여유분.
2) 온도 올리면 호흡↑ = 소모↑. 그런데 이 소모가 있어야 잎·열매가 자란다(소모 없으면 영양기관 성장 정지).
3) 따라서 밤 온도가 곧 배분 다이얼. 낮에 축적한 동화산물을 밤에 얼마나 어디로 쓸지 결정한다.

주간에 식물은 만든 동화산물의 약 2/3를 스스로 소모한다(체온 유지·증산·양분 이동). 이건 사람이 못 건드리는 부분. 사람이 개입하는 건 남은 순광합성분의 밤 배분.

그날 상황별 야간온도 결정

왜 교과서(주25/야10 고정)가 틀렸나

고정 목표값은 그날 벌어들인 에너지를 무시한다. 같은 25/10도라도 맑은 날과 흐린 날의 최적 대응은 정반대. 국내 교과서를 그대로 학습시키면 실제 작물 반응과 어긋난다. 네덜란드식은 한 발 더 나가 "저온·동화산물 많은 날은 오히려 더 낮춰 스트레스를 막으라"고까지 한다. AI는 고정값이 아니라 "상황→대응" 함수를 배워야 한다.

동화산물 추정 실전

동화산물 양은 정밀 계산이 매우 어렵다(변수 큼). 현장은 24시간 평균온도로 대략 추정한다. 더 정밀하게 가려면 증산량 → 순광합성 추정 경로: 광↑ → VPD↓ → 증산↑ → 물 소모↑. 챔버식 등으로 물 출입량을 재 광합성량을 역산한다. AI가 이 추정을 개선하면 = 차별화 포인트. 단, "동화산물 양만으로 열매 크기를 예측할 순 없다"(소모가 변수) — 순광합성으로 봐야 한다.

C

겨울 습도 전투 — 환경제어의 핵심 난제

"온도 오르면 문 열어"로는 스마트팜을 못 쓴다. 경험자들이 가장 힘들어한 부분.

딜레마의 구조

습도가 높으면 빼야 한다. 그런데 환기창을 열면 애써 데운 온도와 밤새 쌓인 CO₂가 같이 빠져나간다 = 에너지·자원 낭비 = 점수 감점. 겨울엔 이 삼각 트레이드오프(습도↓ vs 온도 유지 vs CO₂ 보존)를 미세하게 풀어야 한다.

최소 난방 — 문 안 열고 습도 빼기

핵심 트릭: 실내 온도를 살짝 올리면 상대습도가 떨어진다(같은 수증기량, 포화량↑). 그러면 증산할 공간이 생겨 습도 문제가 풀린다. 환기창은 압력 조절할 틈만 미세하게 연다.

  • 온도 상승 속도 제어: 1시간에 1℃씩만 올린다. 급하게 올리면 결국 온도가 너무 높아져 문을 열어야 하니, 문 안 열고 습도↓·CO₂ 소모·광합성이 되는 지점을 천천히 찾는다.
  • 습도 90% → 80% 정도만 낮춰도 증산이 충분히 일어난다. 목표 습도 대략 75%(수분 덜 뺏기며 증산 여유 20~25% 확보).
아침 시나리오 — 자원을 지키는 타이밍

밤새 호흡으로 CO₂가 700ppm↑ 축적, 습도는 90%↑. 이때 습도 빼겠다고 문 열면 아까운 CO₂까지 날아간다. → 최소 난방으로 습도만 빼고 CO₂는 보존했다가, 해 뜨면 그 CO₂로 광합성. 밤 CO₂를 다 소모할 때까지 환기 안 해도 되는 시점을 찾는 게 고급 제어.

천창 미세제어 + PI 밴드

  • 습기는 온실 상층부에 몰린다(더운 습공기가 위로). 천창을 확 열면 습기+온도 다 빠져 저습이 되고 광합성 속도가 50%로 반토막(기공도 50% 닫힘). 그래서 아주 미세하게 연다.
  • 여닫는 속도·민감도를 조절하는 게 PI(P밴드) 제어. 주기별로 P값을 바꾼다:
주기P값이유
아침 주기높게(둔감)예민하게 반응하면 안 됨. 급변 방지
한낮(11~14시) 고광·고온낮게(민감)적극 환기로 온도 관리
오후 / 야간상황별계절·CO₂ 공급 여부 반영
상호배제 규칙 — CO₂ 공급 중엔 함부로 못 연다

CO₂를 공급하는 중이면 환기창을 열면 CO₂가 같이 배출된다. CO₂ 공급 ↔ 환기는 동시에 하면 낭비. 제어 엔진에 상호배제(mutual exclusion) 규칙으로 넣어야 한다.

조조가온 / 예비가온

해 뜨기 전에 미리 환경을 데워 준비하는 선제 제어. 스펙이 충분한 스마트팜인데도 과하게 열어 에너지를 낭비하는 팀이 많다. 낭비 없이 광합성량을 늘리는 예열 패턴을 AI가 판단하면 큰 가점.

에너지 = 점수

최근 스마트팜은 경영(비용) 최적화가 화두. 성과만 좋게 하려고 난방·환기로 에너지를 펑펑 쓰면 감점. "생산량↑ + 에너지·물질 소모↓"의 동시 최적화가 목표. 시설 스펙(측창 유무, 천창만 있는지, 배기팬 유무 — 농진청 표준은 보통 천창만)에 맞춰 제어를 짜야 한다.

D

관수 · 위험 회피 규칙

스마트팜 하는 사람도 놓치는 함정. 룰베이스로 확실히 막아야 하는 것들.

비 오는 날 관수 — 치명적 실수 1위

"비 오면 증산 안 하니 물 끄자"는 관행이 작물을 죽인다. 흐려도 습도 100%는 아니라 VPD가 0.1~0.5로 형성되면 증산은 일어난다. 물을 안 주면:

· 칼슘 생장점 공급 실패 → 세포 외피 미형성 → 생장점 괴사
· 칼륨 공급 지연 → 기공 개폐 불능 → 세포 파괴

양액기가 누적광량 기준으로만 관수하면, 비·저온으로 광량이 안 차 관수를 건너뛰어 사고. 대책 = 일정 간격 강제 관수 세팅 + AI가 양액기 상태·VPD 감지해 개입.

수정(受精) 방해 요인 — 개화기 체크리스트

꽃이 수정 안 되면 열매 자체가 없다. 개화~비대 사이 반드시 확인:

  • 온도 과다: 작물 좋으라고 올리면 벌이 활동 안 하는 온도가 됨
  • VPD 너무 낮음: 수정 방해
  • 방제 농약: 벌 활동·수정 방해 → 개화기 타이밍 조율
  • 총채벌레: 꽃 속 씨앗(생식기관) 흡즙 → 꽃 망가뜨림. 사람 눈에도 안 보임

병해충 — 환경 패턴으로 예측

환경 조건임계발생
저온 다습습도 94%↑ · 4시간↑곰팡이병 — 잿빛곰팡이·탄저·꽃곰팡이
고온 저습급격한 광↑·습도↓총채벌레·응애 (춘분 이후 대발생)

해충은 0.1mm 이하라 이미지 직접 감지 어렵다. → 흡즙 피해 흔적(잎 얼룩 vs 정상 잎 대조) 학습 + 환경 패턴 기반 발생확률 예측이 현실적. 병 걸리는 환경 자체를 안 만드는 예방이 우선.

E

통합 룰베이스 초안

위 원리를 팀의 룰베이스 모델에 넣을 수 있는 의사코드로. 실제 임계값은 데이터로 튜닝 전제.

쓰는 법

회의에서 나온 대로, 특정 조건 = 특정 결정인 규칙은 룰베이스로 명시하고, 예측(착과수·환경)은 ML 모델(부스팅트리·LSTM)에 맡긴 뒤, 이 룰이 그 위에 안전장치·도메인 지식으로 겹친다. 아래는 골격.

# 1. 야간온도 = 동화산물 배분 다이얼
if 낮_적산광량 == 높음 and 동화산물_추정 == 많음:
    야간온도 ← 올림          # 소모↑, 열매 과비대 방지 + 잎 키움
elif 낮_일사 == 적음:
    온도 ← 낮춤              # 소모 억제, 생리 스트레스 방지
if 야간_급랭_예보:
    주간온도 ← 낮춤          # 밤을 못 올리니 편차를 낮에서 줄임

# 2. 영양 ↔ 생식 저울
if LAI < 2.5 or 신엽출현 == 느림:
    목표 ← 영양생장          # 온·습↑, 함수율↑, VPD 0.8 아래
elif 수확기 or 과영양(잎만_무성):
    목표 ← 생식생장          # 광 스트레스, 온도↓, 함수율↓, VPD 0.8 위
if LAI > 2.5:
    작업 ← 적엽              # 수지 안 맞음
if 겨울 and 일장 < 임계 and 생식_과잉:
    보광 ← ON (≥2000lux)     # 낮 길이 늘려 영양생장 복귀

# 3. 겨울 습도 (문 열기 전에 난방부터)
if 습도 > 85:
    if 겨울 and (CO2 > 700 or 온도 여유):
        최소난방 ← ON        # 상대습도↓, CO2·온도 보존
        온도상승 ← 1℃/시 제한
        천창 ← 미세개방        # 확 열면 광합성 50%↓
    else:
        환기 ← 확대
if CO2_공급 == ON:
    환기 ← 금지/최소          # 상호배제: 같이 하면 낭비
P밴드값 ← 아침:높음 / 한낮:낮음   # 주기별 민감도

# 4. 위험 회피 (안전장치)
if VPD > 0.1:               # 비 와도 증산 가능
    관수 ← 강제_최소_보장      # 칼슘·칼륨 이동, 생장점 보호
if 습도 >= 94 지속 >= 4시간:
    경보 ← 곰팡이_위험
if 급격_광↑ and 습도↓:
    경보 ← 응애·총채_위험
if 개화기:
    온도 ← 벌_활동_범위 유지    # 너무 높이면 수정 실패
    농약방제 ← 개화_타이밍_회피

# 5. 비용 = 점수 (모든 결정 위에)
minimize(에너지_소모) subject to (생산량·품질 목표)
설계 원칙 한 줄

ML은 "이 환경이면 착과 몇 개"를 예측하고, 룰베이스는 "그래서 지금 뭘 해야 안 죽고 비용 아끼며 잘 크나"를 결정한다. 도메인 지식은 후자에 산다.

기본 공부 노트 12장의 심화편. 회의록 1·2·3(2026-07-03)의 전문가 발언을 제어 원리·의사코드로 재구성. 임계값(550W·750ppm·600℃·2.5·94% 등)은 회의 언급값이며 품종·시설·지역에 따라 재튜닝 필요. 의사코드는 설계 골격으로, 실제 임계·우선순위는 데이터 검증 전제.
