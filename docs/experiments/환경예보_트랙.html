환경 예보 트랙 — 앞으로 h시간 뒤

폐루프 제어는 '지금'만 보면 늦습니다. 한파가 오기 전에 대응하려면 "앞으로 환경이 어떻게 변할지"를 알아야 합니다. 시간별 데이터로 h시간 뒤 내부 온도·CO₂를 예보하는 모델을 만들고, 어디까지 믿을 수 있는지 검증했습니다.

핵심 3가지

  • 온도 예보는 성공 — 6시간 뒤 R² 0.86, 12시간 뒤 0.61. 지속성(그대로 찍기)보다 확실히 우세 → 선제 제어에 사용 가능.
  • CO₂ 예보는 실패 — 지속성을 못 이김(R² ≈ 0). CO₂는 외부기상이 아니라 '제어 행동'(환기·시비)에 좌우되기 때문.
  • 역설의 통찰 — CO₂가 예측 안 된다는 건 우리가 직접 조작하는 레버라는 증거. 온도는 '예보해서 대응', CO₂는 '설정값으로 유지'.

Method

방법 — 부스팅 + 시차(lag) feature

딥러닝(LSTM) 대신, 표에 강한 부스팅에 '과거 시점 값'을 넣어 미래를 예측합니다.

입력 (지금까지 아는 것)

내부 온도·CO₂·습도의 과거 시차(1·3·6·12시간 전), 외부 기상(외기온·일사·풍속), 시간대(주기성). 미래 내부값은 안 씀(누수 차단).

기준선 (지속성)

"h시간 뒤도 지금과 같다"고 찍는 순진한 예측. 우리 모델이 이보다 나아야 예보 가치가 있습니다.

Result · 성공

① 내부온도 예보 — 잘 된다

지평(예측 거리)이 멀수록 어려워지지만, 12시간까지도 지속성보다 낫습니다.

내부온도 예보 R² · 예측 지평별. 12시간 뒤도 R² 0.61로 쓸 만함.

CO₂는 외부기상·시간대로 예측되지 않습니다(모델이 지속성도 못 이김). 이유가 분명합니다 — CO₂ 농도는 환기·시비 같은 '제어 행동'과 식물 호흡에 좌우되지, 날씨로 정해지지 않기 때문입니다.
역설의 통찰: "CO₂가 예측 안 된다"는 건 나쁜 소식이 아니라, CO₂가 곧 우리가 직접 돌리는 조작 레버라는 증거입니다. 그래서 온도는 '예보해서 선제 대응', CO₂는 '설정값으로 유지·제어'로 다루면 됩니다.

Integration

제어 규칙에 어떻게 붙나

예보를 룰의 조건(IF)에 넣으면 '반응'이 '선제 대응'이 됩니다.
예: IF 6시간 뒤 내부온도 예보 < 8℃ THEN 지금부터 난방·보온커튼 가동 — 실제 저온이 닥치기 전에 대비.
온도 예보(R² 0.86@6h)는 이 조건을 신뢰성 있게 제공합니다. CO₂는 예보 대신 설정값(700~900ppm) 유지 제어로.
