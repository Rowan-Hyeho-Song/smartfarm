Why a New Track

왜 기존 v1~v3에 바로 못 붙이나

First Model

첫 모델 — 정직한 두 가지 검증

LightGBM 회귀. 같은 조사기간 행이 학습/검증에 쪼개지지 않게 세그먼트 단위로 나눕니다(누수 차단).

계단형 타깃의 함정 — 행 단위 랜덤 분할로 돌리면 '시간블록 암기'만으로 R²가 가짜로 치솟습니다. 반드시 조사기간(세그먼트) 단위로 나눠야 정직합니다.

행 단위 GroupKFold R²는 0.75(MAE 1.03) — 위 표의 세그먼트평균 지표가 조사기간 단위 실질 성능입니다.

착과수 예측 R² — 실전형 forward-chaining(초록)에서 모델(0.76)이 '직전 주값 그대로'(0.76)를 사실상 못 이긴다.

forward-chaining에서 모델(0.763) ≈ 지속성(0.758). 즉 진짜 미래 예측에선 환경·제어 100개 피처가 '지난주 값'이 이미 주는 정보 위에 거의 아무것도 못 보탰습니다. GroupKFold 0.82는 계절 위치를 보간해 얻은 낙관치입니다.

Interpretation

무엇을 말하나 — 기존 결론의 재현

피처 중요도와 성능 격차가 같은 곳을 가리킵니다.

전체 학습 피처 중요도 1등은 외부누적광량(EXTN_ACCMLT_QOFLG) — 계절이 지날수록 쌓이는 값이라 사실상 생육단계(발달) 프록시입니다. 그 뒤로 외부기온·실내CO₂·공급온도·CO₂설정값이 따릅니다. 조절 가능한 제어보다 '지금이 시즌의 어디쯤인가'가 착과수를 설명한다는 뜻입니다.

기존 프로젝트가 밝힌 "착과수는 환경보다 발달단계·자기상관이 지배"가 이 신규 데이터(단일 구역·2022)에서도 독립적으로 재현됐습니다. 서로 다른 온실·시즌·스키마에서 같은 결론 → 그만큼 견고한 신호입니다.

그래서 이 데이터로 할 일

  • 개화수 데이터와 merge — 짝 데이터 BLMNG_CNT(개화수)와 포맷이 동일. STRG_DT·ZONE_NM으로 붙이면 개화(t) → 착과(t+k) 선행 분석이 드디어 가능(발달단계 신호를 직접 만드는 길).
  • 발달단계 feature 추가 — 정식 후 경과주·적산광량·화방 주기를 명시 피처로. 기존 v3가 개체 lag로 얻은 이득을, 여기선 이 신호로 재현.
  • 제어 · 에너지 트랙 — 제어장비 대량(천창·스크린·CO₂·난방 개도율)은 예측이 아니라 본선 '제어 + 에너지 효율'의 근거. 예측 R²보다 이쪽이 이 데이터의 진짜 가치.
  • 단일 구역 한계 인지 — 구역이 1개라 일반화 검증은 시간분할뿐. 다른 구역·연도 데이터가 붙어야 LOGO급 정직성이 확보됨.
단일 구역 · 27개 조사기간 · 계단형 타깃이라 수치는 표본이 작습니다. forward-chaining 19주 평가도 한 온실의 한 시즌일 뿐 — 결론(발달단계 지배)은 방향으로 받아들이고, 구역·연도가 늘면 재확인해야 합니다.
