v3.3 — 발달축 파생 (정보량 최상위, 그러나 과적합)

v3.2가 "환경은 무효, 발달이 축"이라고 알려줬습니다. 그래서 이번엔 발달 상태를 근사하는 파생 6개(누적·템포·소스싱크·자기 궤적상 위치·국면)를 넣었습니다. 결과는 반전 — 정보량은 압도적(중요도 2·3·4위)인데 일반화 이득은 없었습니다.

핵심 결론 3가지

  • 가설은 맞았다 (중요도로 증명) — 발달 파생 6개 중 5개가 전체 47개 중 top 9. 착과수_개체누적률_전주(2위)·착과수_개체누적_전주(3위)가 개체 lag보다도 위. 착과수가 발달 지배임을 다시, 더 강하게 확인.
  • 그런데 일반화는 안 됐다 — 동일 params에서 v3.1 대비 개체 R² 0.856 → 0.848, MAE중앙 0.686 → 0.744(악화). 자체 튜닝으로 회복해도(0.857·0.938·0.716) v3.1의 MAE중앙 0.686을 못 넘음.
  • 병목은 피처가 아니라 데이터 크기 — 신호가 강한 파생일수록 600행·온실 4개 LOGO에선 과적합으로 기웁니다. 아이디어가 아니라 표본 수가 한계. → v3.1 유지, 발달 파생은 데이터 확대 시 재평가.

이제 병목은 아이디어가 아니라 데이터

파생 3연속 실험(v3.1 모멘텀 채택 · v3.2 환경추세 기각 · v3.3 발달축 보류)이 하나의 결론으로 수렴합니다: 600행·온실 4개에서 우리는 이미 피처의 한계 수확 구간에 있습니다. v3.1의 교차항 하나가 준 이득 이후로, 더 많은(심지어 더 강한) 파생은 일반화를 못 올리고 과적합만 키웁니다.

보류. 착과수 생산 모델 = v3.1 유지. 발달 파생은 data_prep.py에 남겨 두고(재현·재사용), 표본이 늘면(더 많은 온실·시즌) 최우선 재평가 대상 — 그때는 강한 신호를 과적합 없이 받아낼 여지가 생깁니다.

전략 전환 신호 — 다음 투자는 '새 파생'보다 데이터 확대(신규 2022 데이터 통합·온실 추가)나 규제·앙상블(작은 데이터에서 과적합 억제) 쪽이 기대값이 큽니다.
