딸기 예측모델 연습환경
설치 & 보유 데이터 준비

이미 확보한 4개 온실의 환경·생육 데이터로 5회 예선(공공데이터 기반 딸기 예측)을 재현하기 위한 준비 문서입니다. 실전에 쓸 맥북에어(Apple Silicon)를 기준으로 설치법을 쓰고, Windows에서도 똑같은 환경을 만들 수 있게 정리했습니다.

범위 예측 모델까지 (예선 재현) 기준 환경 MacBook Air · Windows 동일 구성 데이터 보유 중인 파일만 사용 주력 LightGBM·CatBoost·TabPFN
내가 가진 데이터 두 데이터 합치기 데이터 정제 설치 · 맥북에어 설치 · Windows 동일화 환경 공유 폴더 구조 다음 단계

60초 요약

  • 데이터는 이미 있음 — 새로 받을 것 없이 보유한 생육 1개 + 환경 4개 파일로 예선 형태를 재현합니다. (생육정보만 CP949였고 이미 UTF-8로 변환됨)
  • 핵심 작업 — 시간별 환경(wide) 과 주1회 생육(long) 을 온실번호+날짜로 합치는 것. 생육을 기준 시점으로 두고 환경을 직전 구간으로 집계해 붙입니다.
  • 맥북에어 기준 — brew install libomp → conda 환경 strawberry → LightGBM·CatBoost·TabPFN·pandas. PyTorch는 MPS(Metal).
  • Windows도 동일 — 같은 environment.yml로 재현. 차이는 PyTorch(CUDA) 한 줄과, 맥의 libomp뿐. 코드는 그대로 이식됩니다.

Data

내가 가진 데이터 — 이것만으로 시작

data/ 폴더의 실제 파일 5개. 새로 다운로드할 필요 없습니다.

① 환경정보 · 온실별 4개 파일 입력(X)

24_환경정보-1온실.csv ~ -4온실.csv — 시간별(1시간 간격) 센서 데이터. 온실당 약 3,061~3,085행, 2024-09-29/30 ~ 2025-02-07. 인코딩은 원래부터 UTF-8.

급액EC / 급액PH양액 농도 / 산도1.1 / 6.3

4개 파일은 온실번호 컬럼이 없으니, 읽을 때 파일명에서 온실번호(1~4)를 붙여 한 테이블로 합치세요.

② 생육정보 · 1개 파일 예측 대상(Y) 후보

24_생육정보.csv — 주 1회 조사(총 15일, 2024-11-01 ~ 2025-02-03), 4,201행. long(세로) 포맷이라 한 행이 “한 표본의 한 항목”입니다. CP949였던 걸 UTF-8(BOM)로 변환 완료.

컬럼의미
온실번호 / 조사일자1~4 / 주1회 날짜 (조인 키)
측정라인 / 표본번호같은 온실 내 개체 구분
조사항목 / 단위 / 조사항목값초장·엽장·엽폭·엽병장·엽수·관부직경·착과수 중 하나 + 값
long 포맷이라 그대로는 학습에 못 씁니다. 조사항목을 컬럼으로 펼쳐(pivot) “한 행 = 한 표본의 모든 항목” 형태(wide)로 바꿔야 합니다.

예선 재현 = 이 데이터로 무엇을 예측?

보유 데이터로 만들 수 있는 대표 과제:

트랙입력(X)예측(Y)
환경 예측과거 환경 시계열다음 구간 내부온도·CO₂·누적광량
생육 예측직전 구간 환경 집계 + 현재 생육다음 조사일의 초장·엽수·착과수
연습 1순위로는 “직전 7일 환경 → 다음 조사일 착과수/초장 예측”을 추천합니다. 우리 데이터 구조(주1회 생육 × 시간별 환경)에 가장 자연스럽게 맞습니다.

Step 1

두 데이터 합치기 — 핵심 한 단계

환경(시간별·wide) + 생육(주1회·long) → 학습용 한 테이블.

  1. 환경 4파일 합치기

    파일명에서 온실번호 부여 → 세로로 concat. 측정일시를 datetime으로 파싱(%m/%d/%Y %H:%M).

  2. 생육 pivot (long→wide)

    [온실번호, 조사일자, 측정라인, 표본번호] 기준으로 조사항목→컬럼, 조사항목값→값. 한 표본이 한 행이 됩니다.

  3. 환경을 생육 시점으로 집계

    각 조사일자의 직전 7일 환경을 평균·합·최대/최소로 요약(누적광량=일사 합, 평균온도, 야간최저 등). 이게 생육을 설명하는 feature.

  4. 온실번호+날짜로 조인

    집계 환경 feature를 pivot된 생육에 붙이면 “X(환경 요약) → Y(생육)” 한 테이블 완성.

이 파이프라인을 src/data_prep.py 하나로 만들어 두면, 나중에 새 온실 데이터가 와도 같은 코드로 재생성됩니다. 원하시면 이 스크립트 초안을 만들어 드릴게요.

Step 2

데이터 정제 체크리스트

시계열 예측은 모델보다 이 전처리가 점수를 가릅니다. (우리 데이터에서 실제로 걸리는 것들)

  1. 인코딩 통일

    생육=UTF-8(BOM) 변환됨, 환경=UTF-8. pandas에선 encoding='utf-8-sig'로 읽으면 5개 파일 모두 안전.

  2. 날짜 파싱·정렬

    환경 측정일시는 M/D/YYYY H:MM 형식 명시 파싱. 온실별·시간순 정렬.

  3. 이상치·결측

    내부습도 최대 99.9%·최소 10.2%, CO₂ 최대 1946 같은 스파이크 점검. 물리적 불가값 클리핑, 센서 결측은 시간보간.

  4. 파생 변수

    누적광량(DLI), VPD(온도·습도), 주야 구분(일사>0), 이동평균·차분, 요일/계절 주기.

  5. 데이터 누수 방지

    train/valid는 시간 순으로 분할(랜덤 금지). 생육 예측 시 미래 환경이 X에 새지 않게 “직전 구간”만 사용.

  6. 온실 편차 인지

    4개 온실은 평균 온도·CO₂·EC가 서로 다릅니다(예: CO₂ 349~458). 온실ID를 feature로 넣거나 온실별 검증.

Step 3 · 기준 환경

MACBOOK AIR · Apple Silicon

설치 (실전 기준)

CUDA가 없고 팬리스·통합메모리라, LightGBM·CatBoost·TabPFN 같은 가벼운 정형 모델이 주력입니다. PyTorch는 MPS(Metal) 백엔드.

1) Homebrew & libomp 필수

맥에서 LightGBM은 libomp가 없으면 import부터 실패합니다. 반드시 먼저.

# Homebrew 없으면 설치
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

# 필수 도구 + LightGBM 의존성 + Miniforge(conda)
brew install libomp git miniforge

2) 가상환경 & 라이브러리

conda create -n strawberry python=3.11 -y
conda activate strawberry

# PyTorch — Apple Silicon (MPS/Metal, CUDA 아님)
pip install torch torchvision

# 코어 (데이터·시각화·노트북)
pip install numpy pandas polars scikit-learn matplotlib seaborn jupyterlab

# 예측 주력 — 부스팅 트리 + 튜닝
pip install lightgbm xgboost catboost optuna

# 시계열 프레임워크
pip install darts sktime statsmodels

# 정형/시계열 파운데이션 (CPU/MPS로 동작)
pip install tabpfn chronos-forecasting
확인: python -c "import lightgbm, torch; print(torch.backends.mps.is_available())" — 에러 없이 True(또는 False라도 import만 되면)면 준비 완료.
맥북에어는 “추론·제출” 중심으로. 무거운 시계열 파운데이션 학습·대규모 튜닝은 부담되니, 필요하면 Windows(GPU)에서 학습하고 학습된 모델만 맥으로 옮겨 예측·제출에 쓰는 분담을 권장합니다. 정형 모델(LightGBM/CatBoost/TabPFN)은 맥에서도 충분히 돕니다.

Step 4 · 같은 환경

WINDOWS

동일 환경 만들기

맥과 같은 conda 환경 이름·같은 패키지로 맞춰, 코드가 양쪽에서 그대로 돌게 합니다. 차이는 딱 두 줄.

맥과 동일하게 설치

# Miniforge / Git (winget)
winget install CondaForge.Miniforge3
winget install Git.Git

conda create -n strawberry python=3.11 -y
conda activate strawberry

# ★ 차이 1: PyTorch는 CUDA 빌드 (GPU 있을 때)
pip install torch --index-url https://download.pytorch.org/whl/cu124
#   GPU 없으면 그냥:  pip install torch

# 이하 전부 맥과 동일
pip install numpy pandas polars scikit-learn matplotlib seaborn jupyterlab
pip install lightgbm xgboost catboost optuna
pip install darts sktime statsmodels
pip install tabpfn chronos-forecasting
# ★ 차이 2: Windows는 libomp brew 설치 불필요 (LightGBM 바로 됨)

Sync

두 컴퓨터 환경 똑같이 유지하기

손으로 맞추지 말고 파일 하나로 재현하세요.

# 한쪽에서 환경 내보내기 (torch 등 OS별 라인은 빼고 관리 권장)
pip freeze > requirements.txt

# 다른 컴퓨터에서 그대로 재현
conda create -n strawberry python=3.11 -y && conda activate strawberry
pip install -r requirements.txt   # torch는 위 OS별 명령으로 따로
PyTorch 한 줄만 OS별로 다르게(맥=기본, 윈=CUDA) 두고, 나머지는 requirements.txt로 공유하면 양쪽이 항상 같은 버전으로 맞습니다. Git으로 src/ 와 함께 올려두세요.
