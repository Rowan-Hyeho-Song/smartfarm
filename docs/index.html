신규 테스트 데이터(2022, 103열)로 예측·처방·결과지 3트랙을 검증 — '착과수는 발달단계 지배·제어는 상관≠인과'라는 결론이 다른 온실·시즌에서 재현됐습니다.

파생 엔지니어링 트랙(v3.1~v3.4) — 해커톤식 '대량 파생'을 그대로 따르지 않고 LOGO로 걸러: 모멘텀 교차항은 채택(MAE중앙 0.73→0.69), 환경추세는 기각(발달 지배 재확인), 발달축은 정보량 최상위였지만 과적합→보류, 규제·앙상블(LGBM+CatBoost+Ridge)로도 v3.1 못 넘음. 결론: 이제 병목은 피처가 아니라 데이터 크기 — 다음 가치는 예측 밖(제어·불확실성·실증)에 있습니다.

v3.1 — 모멘텀(증가속도) 파생 추가

실험 · v3.2

v3.3 — 발달축 파생 (과적합·보류)

실험 · v3.3

Under the Hood

파이프라인 & 스택

data_prep → train / advanced / optuna / train_agg(v2) → conformal → predict(v2) → forecast_env

언어 Python 3.11 모델 LightGBM · CatBoost 튜닝 Optuna 불확실성 Conformal (CQR) 검증 온실별 LOGO · 시간분할 데이터 온실 1~4 · 2024-11~2025-02
정직성 원칙 — 모든 성능은 '처음 보는 상황'을 흉내 낸 검증(LOGO/시간분할) 기준이며, 데이터 누수를 구조적으로 차단했습니다. v2.1(집계 R² 0.89)·v3(개체 추적, 집계 0.94·개체 0.85·MAE중앙 0.73) 모두 온실 4개·소표본 기준이라 데이터 확대 시 재확인이 필요합니다.
